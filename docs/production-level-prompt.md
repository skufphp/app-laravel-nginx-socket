# ✅ Production Readiness Technical Audit Prompt

> Проанализируй данный boilerplate-проект как senior software architect и технический аудитор.
>
> Оцени его готовность к разработке production-уровня приложения по шкале от 1 до 10.
>
> Сформируй результат в виде структурированного технического отчёта, который можно напрямую передать другому AI-агенту для доработки проекта.
>
> Проведи анализ по следующим направлениям:
>
> * Архитектура и структура проекта
> * Масштабируемость
> * Чистота и поддерживаемость кода
> * Безопасность
> * Производительность
> * DevOps (Docker, CI/CD, окружения, конфигурации)
> * Тестирование
> * Логирование и мониторинг
> * Developer Experience
>
> Представь, что проект:
>
> * должен выдерживать 1000+ одновременных пользователей
> * рассчитан на долгосрочную поддержку
> * должен быть безопасным для production-среды
>
> В отчёте обязательно предоставь:

---

### 1. Executive Summary

Краткое общее заключение о состоянии проекта.

### 2. Итоговая оценка (X/10)

Обоснуй оценку с точки зрения production-готовности.

### 3. Сильные стороны проекта

Перечисли, что реализовано правильно.

### 4. Основные проблемы и риски

Укажи:

* архитектурные слабости
* потенциальные точки отказа
* уязвимости
* технический долг
* ограничения масштабируемости

### 5. Production Gap Analysis

Чётко определи, чего не хватает проекту до production-уровня.

### 6. Приоритетный план доработки (Task List для AI-агента)

Сформируй конкретные задачи в формате:

* **Задача**
* **Почему это важно**
* **Как исправить (конкретные технические рекомендации)**
* **Приоритет: High / Medium / Low**
* **Ожидаемый результат**

---

❗ Требования к стилю ответа:

* Не давай общих советов
* Не пиши абстрактных рекомендаций
* Только конкретные технические улучшения
* Будь максимально критичным
* Пиши так, как будто отчёт отправляется CTO
